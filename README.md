# Инструкция по начальной настройке конфигурации «Управление IT-фирмой для компании «Ваш компьютерный мастер»

Данная инструкция предназначена для специалистов заказчика и администраторов системы. Выполните указанные шаги, чтобы начать использование новой функциональности в информационной базе, в которой уже ведется реальный учёт.

------
 
## 1. Назначение профилей групп доступа

Для обеспечения информационной безопасности и разделения полномочий назначьте пользователям соответствующие профили доступа:

- Специалист: Назначается специалистам, выполняющим обслуживание клиентов. Создание документов «Обслуживание клиента», просмотр личных начислений;
- Менеджер: Назначается сотрудникам отдела по работе с клиентами. Создание и редактирование документов «Обслуживание клиента»;
- Бухгалтер ИТ-фирмы: Назначается сотрудникам бухгалтерии, работающим с первичной документацией. Работа с документами покупки-продажи, доступ к массовому формированию закрывающих документов по абонентским договорам;
- Кадровик-расчетчик: Назначается кадровику, главному бухгалтеру или сотруднику, занимающемуся расчетом заработной платы и планированием отпусков работников. Полный цикл расчета зарплаты, начисление премий, планирование и анализ отпусков.

------

## 2. Настройка оповещений в Telegram

Система поддерживает автоматическое уведомление специалистов о назначенных работах. Для активации выполните следующие действия:

1.	Создание бота: Через бота @BotFather создайте нового технического бота и получите его уникальный API-токен.
2.	Подготовка группы: Создайте общую группу в Telegram и добавьте туда бота в качестве администратора.
3.	Получение ID: Узнайте идентификатор (ID) созданной группы (например, через бота @myidbot).
4.	Логика работы: Оповещения будут приходить в группу автоматически при создании нового документа «Обслуживание клиента», а также при изменении Даты или Специалиста в уже существующем документе.

------

## 3. Заполнение ключевых констант

Для корректной работы автоматизированных алгоритмов и Telegram-бота заполните настройки в разделе «Константы»:

1.	Номенклатура работы специалиста: Укажите услугу, которая используется для автоматического подбора выполненных работ в акты реализации.
2.	Номенклатура абонентская плата: Укажите услугу для ежемесячного фиксированного обслуживания.
3.	Токен управления телеграмм ботом: Введите полученный API-токен бота.
4.	Идентификатор группы для оповещения: Введите полученный ID группы Telegram.

------

## 4. Настройка графиков и параметров расчета

Перед началом расчетов необходимо актуализировать нормативно-справочную информацию:

1.	Графики работы: В справочнике должен быть создан актуальный график (например, «Пятидневка»).
2.	Заполнение календаря: В регистре сведений «Графики работы» должны быть данные о рабочих часах на текущий год. Без этого расчет Оклада невозможен.
Совет: Используйте специализированную обработку «Заполнение графиков» для автоматического формирования записей в регистре.
3.	Условия оплаты: Для всех сотрудников в регистре «Условия оплаты сотрудников» должны быть установлены актуальные размеры окладов и процентов премий.

------

## 5. Порядок начала работы и правила ввода данных

Для корректного формирования отчетности и работы механизмов вытеснения соблюдайте следующие правила:

- Ввод начальной задолженности: Для фиксации долга перед сотрудником на момент запуска используйте документы «Начисление зарплаты» или «Начисление фиксированной премии» за предыдущий период.
- Контроль уникальности периодов: При заполнении документа «Начисление зарплаты» не допускается создание дублирующих строк для одного сотрудника на один и тот же период по видам расчета «Оклад» и «Отпуск» одновременно.
Пример: Нельзя вводить обе строки на период 01.01.2026 – 31.01.2026. В таком случае Оклад будет начислен полностью без учета вытеснения. Отпуск должен вводиться строго на даты фактического отсутствия.
- Выплата зарплаты: Документ «Выплата зарплаты» автоматически контролирует наличие накопленного долга и не позволяет выплатить сумму сверх остатка.
- Планирование отпусков: Внесите данные через документ «График отпусков». Это активирует проверку лимита в 28 дней и визуализацию на Диаграмме Ганта.

------

## 6. Аналитическая отчетность

Пользователям доступны следующие отчёты в подсистеме «Добавленные объекты», а также в подсистемах «Обслуживание клиентов» и «Расчет управленческой зарплаты»:
- Анализ выставленных актов (контроль соответствия работ и реализаций).
- Выработка специалистов (анализ эффективности и начисленных премий).
- Расчеты с сотрудниками (сводная ведомость по начислениям и выплатам).
- Расход запланированных отпусков (сравнение плана и факта).
